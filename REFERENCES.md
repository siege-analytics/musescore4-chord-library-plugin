## Musicians & Pedagogues

- **Ted Greene** (1946-2005) — V-System voicing classification, *Chord Chemistry* (1971, Alfred Publishing), tip barring technique, fingerstyle chord melody.
  - V-System materials: https://www.tedgreene.com/teaching/v_system.asp
  - Chords archive: https://tedgreene.com/teaching/chords.asp
- **George Van Eps** (1913-1998) — 7-string guitar pioneer, diagonal barre technique, harmonic mechanisms approach.
- **Joe Pass** (1929-1994) — Economical chord voicing approach, three-shape system.
  - Pass, Joe, and Bill Thrasher. *Joe Pass Guitar Style.* Gwyn Publishing, 1971.
  - Pass, Joe. *The Joe Pass Guitar Method.* Chappell & Co., 1977 (distributed by Hal Leonard, HL00347734). Source for the six-Chord-Forms positional system (p. 4), the three- to four-note voicing-with-voice-leading principle (p. 30, head note to "Blues for Nina"; voice-leading panel p. 31), and the dominant-alteration/tritone substitution lattice (p. 25 head note to "Joe's Blues"; p. 30). PDF not redistributed (commercial copyright).
- **Paul Vachon** — "How to Read Ted Greene Chord Diagrams." [PDF](https://tedgreene.com/images/lessons/students/PaulVachon/HowToReadTedGreeneChordDiagrams.pdf)
- **James Hober** — V-System mathematical formalization. [PDF](https://tedgreene.com/images/lessons/v_system/02_V-System_Introduction.pdf)
- **Marten Falk** — Guitar teacher at Lilla Akademien; fingering rules used by Norman & Grozman.

## Open-Source Projects & Databases

### Chord Databases with Fingering Data

- **tombatossals/chords-db** — MIT license. Guitar chord database with `fingers` field, barre detection, npm package. https://github.com/tombatossals/chords-db
- **szaza/guitar-chords-db-json** — MIT license. 99,230 guitar chords in JSON with fingerings. Derived from T-vK/chord-collection. https://github.com/szaza/guitar-chords-db-json
- **T-vK/chord-collection** — Source data for szaza's collection. https://github.com/T-vK/chord-collection
- **fusionprogguy/Fretboard** — Python, ~300 chords with Ted Greene V-System classification in CSV. https://github.com/fusionprogguy/Fretboard
- **Giancarlo Facoetti / fachords.com** — UCI/Kaggle Guitar Chords Finger Positions dataset. 2,633 chords, CC BY 4.0. https://archive.ics.uci.edu/dataset/575/guitar+chords+finger+positions

### Fingering Algorithms

- **pcorey/chord** — Elixir. Recursive sieve with 4 constraint rules, barre detection. By Pete Corey. https://github.com/pcorey/chord
- **hyvyys/chord-fingering** — JavaScript. Generates fingerings with difficulty score and barre detection. https://github.com/hyvyys/chord-fingering
- **senshu/TablaZinc** — MiniZinc + JavaScript. Formal constraint solver (Gecode backend) for tablature. https://github.com/senshu/TablaZinc
- **natecdr/tuttut** — Python (PyPI). HMM + Viterbi for melody fingering. https://github.com/natecdr/tuttut
- **burksbuilds/guitar-robot** — Python. A* search with acceleration-based cost function. By Andrew Burks. https://github.com/burksbuilds/guitar-robot
- **RidwanSharkar/Fretboard-Explorer** — React/JS. Backtracking algorithm, 5-fret max. https://github.com/RidwanSharkar/Fretboard-Explorer
- **jonthysell/Chordious** — C# (.NET). Fretboard diagram generator, MIT license. https://github.com/jonthysell/Chordious
- **rlk/picker** — JavaScript. Calculates and renders fretted instrument fingerings. https://github.com/rlk/picker
- **hybridpicker/fretboard-position-finder** — Python/Django. Position finder for scales, arpeggios, chords. https://github.com/hybridpicker/fretboard-position-finder
- **srviest/SoloLa** — Python. Guitar solo transcription with fingering arrangement. https://github.com/srviest/SoloLa

### File Format Libraries

- **Perlence/PyGuitarPro** — Python library for reading/writing Guitar Pro files (GP3-GP5). https://github.com/Perlence/PyGuitarPro

### File Formats — Import Considerations

- **iReal Pro** — `irealb://` URL scheme with obfuscated chord data. Implemented in `IRealParser.js` (#149). Also exports HTML files containing the URL.
- **Band-in-a-Box** — Proprietary binary formats: `.SGU` (song), `.MGU` (melody+guitar). No public specification. PG Music has not published a file format spec. Viable import paths: (1) BiaB text export, (2) BiaB MusicXML export → parse chords, (3) reverse-engineer binary format. MuseScore natively imports MusicXML, so plugin value-add would be extracting chord charts for the walkthrough. See #159.
- **GuitarPro** — `.gp3`/`.gp4`/`.gp5`/`.gpx`/`.gp` formats. MuseScore 4 natively imports these — no plugin work needed.
- **MusicXML** — W3C standard. MuseScore 4 natively imports. Chord symbols in `<harmony>` elements.
